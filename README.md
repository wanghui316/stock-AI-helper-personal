## 项目定位

这是一套为个人A股投资者设计的量化决策辅助系统，而非机构级交易平台。

它不替你做交易决策，而是在你犹豫时，用规则和数据提醒你；在你冲动时，念出你为自己设定的铁律。

**它负责算分，你负责拍板。**


## 设计思路

散户投资中最常见的两个问题是：**买了不知道什么时候卖，跌了不知道要不要扛。**

这套系统的设计目标就是通过一套可重复的量化规则，把“感觉”转化为“分数”，把“犹豫”转化为“规则”。

### 三层决策逻辑

| 层次 | 回答的问题 | 权重 |
|------|-----------|------|
| **安全边际** | 这只票贵不贵？方向对不对？ | 65%（估值+政策+RPE） |
| **择时信号** | 现在买还是等？ | 35%（趋势+量价） |
| **仓位控制** | 买多少？ | 分类上限管理 |

### 核心原则

- **PE定方向，趋势定时点。** PE决定“该不该留”，趋势和量价决定“什么时候出”
- **仓位是第一风控。** 单只股票仓位不超过对应类型上限，月末检查D级票占比
- **问题仓优先处理。** 不符合选股三条红线的持仓，不以“等回本”为由死拿
- **数据缺失时宁可不做，绝不凭空捏造。** 这是最重要的底线


## 核心规则

### 选股三条红线（缺一不可）

1. **政策方向对**：对标“十五五”国家战略性新兴产业和未来产业方向
2. **业绩真实**：净利润为正，PE(TTM)为正
3. **估值合理**：PE(TTM)不超过对应类型合理区间上限

### 三维分类体系（Ⅰ/Ⅱ/Ⅲ型）

| 类型 | PE合理区间 | 仓位上限 | 止损阈值 |
|------|-----------|---------|---------|
| Ⅰ型（周期/资源） | ≤20倍 | ≤10% | -5% |
| Ⅱ型（成长/制造） | 25-40倍 | ≤15% | -8% |
| Ⅲ型（尖端科技） | ≤60倍（有条件） | ≤5% | -10% |

**Ⅲ型放宽条件（需同时满足）：** 营收增速>30% + 经营现金流为正 + 毛利率稳定/提升

### 五维权重评分

| 评价维度 | 权重 |
|---------|------|
| 估值合理性 | 40% |
| 趋势状态 | 20% |
| 政策契合度 | 15% |
| 量价动能（双二阶） | 15% |
| 板块相对估值比（RPE） | 10% |

### 评级标准

| 得分 | 评级 | 操作 |
|------|------|------|
| ≥80 | A级（强烈关注） | 优先加仓/建仓 |
| 70-79 | B级（值得关注） | 建观察仓，等待确认 |
| 60-69 | C级（观望） | 纳入备选池，暂不操作 |
| <60 | D级（回避） | 不建仓，已持仓优先减/清 |

### 双二阶量价共振信号

| 信号 | 含义 | 操作 |
|------|------|------|
| 🟢共振做多 | 量价同时加速 | 最强买入/持有信号 |
| 🟡量热价冷 | 量能加速但价格减速 | 警惕追高 |
| 🟡价热量冷 | 价格加速但量能减速 | 不加仓 |
| 🔴共振减速 | 量价同时减速 | 最强卖出/回避信号 |


## 数据获取

- **雪球（pysnowball）**：实时行情、TTM PE（自动获取）
- **baostock**：历史K线（趋势和量价计算）
- **人机交互**：PE数据自动获取失败时的备用输入

> 代码中已内置雪球Token，开箱即用


## 技术栈

- Python 3.14+
- pandas / numpy（数据处理）
- pysnowball（雪球数据接口）
- baostock（历史K线接口）
- openpyxl（Excel输出）


## 使用步骤

```bash
# 1. 安装依赖
pip install pysnowball baostock pandas openpyxl

# 2. 运行程序
python stock_analysis.py

# 3. 按提示输入PE值（自动获取失败时），获得Excel评分报告
```
